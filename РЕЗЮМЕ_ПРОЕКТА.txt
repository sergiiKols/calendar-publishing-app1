СЕРВИСЫ (3):

1. calendar-app (Next.js приложение)
   • Образ:    Собственный (из Dockerfile)
   • Порт:     3000 (внешний)
   • Функция:  Веб-интерфейс + API
   • Зависит:  от calendar-db

2. calendar-db (PostgreSQL 15)
   • Образ:    postgres:15-alpine
   • Порт:     5433 (внешний) → 5432 (внутренний)
   • Функция:  База данных
   • Volume:   calendar-db-data (персистентный)
   • Авто-инициализация: schema.sql при первом запуске

3. calendar-cron (Cron jobs)
   • Образ:    alpine:latest
   • Функция:  Автопубликация каждые 15 минут
   • Зависит:  от calendar-app

СЕТЬ:
   • Название: calendar-network
   • Тип:      bridge (изолированная)

VOLUMES:
   • calendar-db-data - Персистентное хранилище PostgreSQL

НА СЕРВЕРЕ ВЫПОЛНИТЕ:

   1. cd calendar-app

   2. cp .env.docker.example .env

   3. nano .env
      # ОБЯЗАТЕЛЬНО измените:
      # - POSTGRES_PASSWORD
      # - NEXTAUTH_SECRET (сгенерируйте: openssl rand -base64 32)
      # - CALENDAR_API_KEY
      # - CRON_SECRET

   4. docker-compose up -d

   5. Откройте в браузере: http://ваш-сервер:3000

   6. Создайте админа: http://ваш-сервер:3000/api/reset-admin

   7. Войдите: http://ваш-сервер:3000/login
      Username: admin
      Password: admin123

   Мониторинг:        docker stats

   Backup БД:         docker-compose exec postgres pg_dump \
                      -U calendar_user calendar_db > backup.sql

   Restore БД:        cat backup.sql | docker-compose exec -T \
                      postgres psql -U calendar_user calendar_db

   Обновить:          git pull && docker-compose down && \
                      docker-compose build --no-cache && \
                      docker-compose up -d
